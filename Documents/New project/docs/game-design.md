# 多人養成討伐原型進度

這份文件描述目前版本「實際已存在」的玩法、系統與頁面，不寫純想像中的未來規格。

## 何時更新這份文件

只要有以下任一變化，就要同步更新：

- 新增或刪除功能
- 修改遊戲規則、限制、獎勵
- 改變主要頁面內容
- 新增新的系統分頁

## 目前已完成

- 全站主要介面改成中文
- 左側固定導覽
- 主內容改成切頁式，不需要長頁捲動
- 行動改成單一隊列系統
- 房間戰鬥切到獨立戰鬥畫面
- 新增通知、戰報、好友、簽到
- 新增陣營、城池、外交、公庫、玩家市場
- 新增鍛造、裝備耐久、材料與玩家自訂排序
- 新增 Admin 管理工具、公告、每日獎勵與突發活動設定
- 介面已開始往 RPG dashboard / HUD 方向整理，導覽加入 icon 與更明確的面板層次

## 角色頁

目前角色頁可看到：

- HP / MP / 精力
- 金錢 / 材料
- 攻擊 / 防禦 / 運氣 / 智慧 / 韌性等能力值
- 背包與裝備
- 稱號與狀態欄
- 職業與職責配置
- 每日簽到 / 突發活動狀態

## 行動系統

目前規則：

- 同一時間只會執行一件事
- 可以先排後續隊列
- 隊列總長度上限為一天
- 行動期間仍可建立或加入隊伍
- 角色只要有任何隊列項目，就不算閒暇
- 在線完成的行動獎勵 +20%

目前狀態顯示：

- 閒暇中
- 訓練中
- 挖礦中
- 休息中
- 鍛造中
- 狩獵中

進攻、狩獵、移動這類隊伍行動必須由隊長發起，而且隊伍內所有成員都必須是閒暇中。

目前可排的行動：

- 訓練：
  - 釣魚
  - 跳繩
  - 讀書
  - 伏地挺身
  - 沉思
  - 拳擊
- 恢復：
  - 休息
- 挖礦：
  - 淺層礦坑
  - 深層礦坑

## 戰鬥系統

目前流程：

- 建立房間
- 加入房間
- 房主開始討伐
- 可隨時組隊，但開始討伐時會檢查全隊是否閒暇
- 自動戰鬥逐 tick 推進
- Boss 自動攻擊
- 戰鬥結束後寫入戰報
- HP / MP / 精力結果會回寫到角色資料

目前支援：

- 一般討伐房
- 攻城相關戰鬥紀錄
- 詳細戰報檢視
- 依職責進行簡易戰鬥邏輯分流

## 陣營系統

目前已有：

- 陣營加入
- 城池地圖
- 合作外交
- 宣戰
- 公庫分配
- 攻城
- 玩家市場可見性與陣營關聯

攻城規則：

- 沒有隊伍時，角色必須閒暇中才能單人攻城
- 有隊伍時，只有隊長可以發起攻城
- 隊伍攻城前，所有隊員都必須閒暇中
- 隊伍攻城會記錄多名參與者並消耗參與者 HP / MP / 精力

- 本地持久化：`server/server/data/store.json`

- 大部分系統資料：`server/src/persistence/localStore.ts`
