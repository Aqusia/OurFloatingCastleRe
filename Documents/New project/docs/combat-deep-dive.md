# 戰鬥系統深入參考 (Combat Deep Dive)

本文件完整記錄《Our Floating Castle》的戰鬥機制與公式，作為 `docs/game-design.md` 戰鬥段的延伸技術參考。所有公式以 `server/src/combatEngine.ts`、`server/src/game.ts`、`server/src/persistence/localStore.ts` 為準，並由 `balance-sim/` 的 Python 模擬重新驗證。

## 1. 連擊鏈（SAO 式）

- **連擊上限 = 戰鬥等級**（硬上限 20）：`comboCapForLevel(battleLevel) = max(1, min(20, floor(battleLevel)))`。
- **續擊率**：`min(0.94, 0.46 + 技巧×0.009 + 速度×0.014 + 戰鬥等級×0.006)`。第 2 擊起每擊消耗 1 點行動資源（法師吃 MP，其餘吃精力），資源耗盡則中斷。
- **落空率**：`max(0.015, 0.11 − 技巧×0.003 − 速度×0.004)`，任一擊落空即中斷。
- **逐擊暴擊**：暴擊率 `min(0.42, 0.05 + 技巧×0.008 + 運氣×0.004 + 速度×0.002)`；暴擊倍率 `1.45 + min(0.65, 運氣×0.01 + 智慧×0.003)`。
- **單擊基準傷害**（依職業）：戰士 `8 + 攻×1.5 + 技×0.5`；刺客 `7 + 攻×1.2 + 技×1.0 + 運×0.4`；法師 `8 + 智×1.8 + 技×0.5`；補師 `6 + 攻×0.35 + 速×0.8 + 智×0.75`（皆加 `floor(等級/2)` 等級加成）。
- **逐擊衰減**：第 i 擊傷害 `× max(0.4, 0.9^(i−1))`，再乘 0.9–1.1 隨機浮動。
- **終結技**：連擊達 4 段以上收尾時觸發，爆發傷害並把 Boss 下一輪攻勢壓到 0.88×。

### 1.1 連擊里程碑（2026-06-23 新增）

連擊長度達門檻時，整段連擊追加爆發傷害（加在最後一擊上，維持「總傷害＝各擊總和」的不變式）：

| 連擊段數 | 追加傷害 |
| --- | --- |
| 8 | +12% |
| 12 | +20% |
| 16 | +30% |
| 20 | +40% |

只取達到的最高門檻計算一次，並發出 `combo_milestone` 戰報事件。

## 2. 戰術事件（單次觸發）

| 事件 | 觸發來源 | 效果 |
| --- | --- | --- |
| 精準破防 armor_break | 技巧 + 智慧/2 | Boss 下次攻勢 ×0.9 |
| 隊友支援 ally_support | 低血隊友 + 運氣/技巧 | 回復隊友生命 |
| 幸運事故 lucky_twist | 運氣 | Boss 額外受創 + 攻勢 ×0.94 |
| 危急閃避 danger_dodge | 低血(<38%) + 運氣/速度/技巧 | 減免 45% 傷害 |
| 格擋減傷 block | 防禦/技巧/韌性 | 減免 22–55% 傷害 |
| 技巧反擊 counter_strike | 技巧/速度 | 對 Boss 追加傷害 |
| Boss 反制 boss_counter | tick≥2 | 對全隊施加壓力傷害 |

## 3. 進階戰技與狀態效果（2026-06-23 新增）

### 3.1 進階攻擊戰技（`rollOffensiveSpecials`）

| 戰技 | 觸發來源（chance base） | 效果 |
| --- | --- | --- |
| 破甲穿刺 piercing_strike | 攻擊 + 技巧/2（0.05） | 追加 `40%×單擊基準` 爆發，並施加「破甲」 |
| 灼燒 ignite | 智慧 + 技巧/2（0.05） | 施加「燃燒」（DoT = `15%×單擊基準`，3 回合） |
| 劇毒 poison_strike | 運氣 + 技巧（0.045） | 施加「中毒」（DoT = `10%×單擊基準`，3 回合，可疊加） |
| 冰封 frostbite | 智慧 + 速度（0.04） | 施加「冰凍」（Boss 下次攻勢 ×0.6，1 回合） |
| 震懾 stun_blow | 攻擊 + 技巧（0.03） | 施加「眩暈」（Boss 跳過下次行動，1 回合） |

### 3.2 狀態結算（`tickStatuses`，每回合開頭）

| 狀態 | 種類 | 結算效果 |
| --- | --- | --- |
| 燃燒 burn | DoT | 每回合對 Boss 造成 `magnitude` 傷害 |
| 中毒 poison | DoT（可疊加，上限 5 層強度） | 同上，疊加更高 |
| 冰凍 freeze | 減攻 | Boss 該回合攻擊 × `magnitude`(0.6) |
| 眩暈 stun | 行動干擾 | Boss 該回合跳過攻擊 |
| 破甲 armor_break | 受擊加重 | 玩家該回合對 Boss 傷害 ×(1 + magnitude) |

狀態以 `applyStatuses` 併入：中毒疊加強度，其餘刷新持續時間並取較高強度。每回合遞減 `remaining`，歸零則移除。

## 4. Boss 狂怒階段（2026-06-23 新增）

當 Boss 血量低於 `enrageThreshold`（預設 40%）時進入狂怒：攻擊力 ×1.25，發出 `boss_enrage` 事件，整場僅觸發一次（探險模式每隻怪獨立判定）。

## 5. 減傷與敵人縮放

- **受傷減免**：`防禦/(防禦+80)` 百分比 + `floor(韌性/4) + floor(防禦/10)` 固定，最低 1。
- **升級需求**：`max(60, 70 + 等級^1.5 × 40)`。
- **敵人血量**：`(基礎 + 層×18) × (1 + lf×0.22 + lf²×0.012)`（lf = 戰鬥等級−1），二次項避免高等級被秒殺。
- **敵人攻擊**：`(基礎 + 層×3) × (1 + lf×0.08)`。

## 6. 平衡驗證

所有上述機制皆在 `balance-sim/` 以 Python 重新實作並做 Monte Carlo 模擬（每格 4000 場、固定種子）。重點結論：

- 最大連擊數 = 戰鬥等級、終結率隨等級上升（0%→~35–55%）已驗證。
- 豐富化後因傷害來源更多元（DoT 吃智慧、破甲吃攻擊…），各職業勝率落差較舊版收斂；戰士不再於菁英難度墊底崩盤。
- 戰士基礎技巧 4→8、速度 5→6 的調整在新機制下仍位於落差近最小區（網格搜尋驗證）。

詳細方法學見 `docs/balance-methodology.md` 與 `balance-sim/README.md`。
